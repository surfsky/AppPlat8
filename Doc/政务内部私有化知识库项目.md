# 政务内部私有化知识库项目设计方案

适用场景：**政务内网 / 专网私有化部署、内部办公、几百人规模、等保合规、涉密可控**

***

## 一、整体定位

* 部署环境：**政务内网 / 政务外网隔离环境，纯私有化，无外网调用**
* 用户规模：区级 / 局级单位，几百人，多部门、多层级
* 核心目标：**内部政策法规、业务流程、办事指南、会议材料、数据资料统一管理 + 安全可控的 AI 问答辅助**
* 合规要求：等保 2.0、操作留痕、不可外传、可审计、可追溯

***

# 二、授权与权限体系（政务版严格版）

## 1. 组织架构（贴合政务体系）

* 单位（租户）
* 科室 / 处室
* 岗位
* 人员
* 临时工作组（专项工作、专班）

## 2. 权限模型：RBAC + 数据分级 + 岗位权限

严格采用**三员分立**（管理、安全、审计）满足政务监管要求

### （1）三员角色（必设）

1. **系统管理员**： 用户管理、机构管理、资源配置、系统监控
2. **安全保密员**： 密级配置、权限审核、水印策略、外发控制
3. **审计管理员**： 查看全量日志，不可操作系统，仅审计

### （2）业务角色

* 单位领导：全局查阅、数据看板
* 科室负责人：本部门知识库管理、授权
* 业务经办人：上传、编辑、查阅本部门资料
* 普通人员：只读
* 涉密岗位：仅可访问对应密级文档
* 临时人员：限时、限范围访问

## 3. 文档密级与可见范围（政务核心）

* 公开（对内公开）
* 内部
* 秘密
* 机密
* 绝密（按需启用）

控制规则：

* 低密级不可看高密级
* 同密级按部门隔离
* 涉密文档单独授权，**不继承、不自动扩散**
* 涉密文档禁止 AI 总结 / 禁止 AI 问答（可开关）

## 4. 细粒度操作权限


* 查看
* 在线预览
* 编辑
* 上传
* 删除 / 归档
* 下载
* 打印
* 分享（仅限内部）
* 复制内容
* 使用 AI 问答
* 使用 AI 总结

## 5. 安全与审计（政务必备）

* 登录：统一身份认证（政务统一账号 / AD 域 / SSO）
* 全程操作日志：谁、何时、哪台终端、看了什么、复制了什么、下载了什么
* 预览水印：姓名 + 科室 + 时间 + 脱敏 ID
* 禁止外网转发、禁止外链分享
* 下载二次审批（可选）
* 异常访问自动告警（频繁下载、异地登录）

***

# 三、AI 能力设计（私有化、无外网、政务安全版）

全部基于**本地私有化大模型**，不调用任何外部 API，数据不出内网。

## 1. 基础 AI 能力

* 文档解析：PDF、Word、图片、扫描件 OCR
* 自动抽取：文号、主题、关键词、责任科室、时效
* 自动生成目录、章节结构化
* 自动摘要（精简版 / 详细版）
* 内容校对、格式规范化

## 2. 核心 RAG 智能问答（内部助手）

* 基于政策文件、办事流程、会议纪要问答
* 多轮对话，上下文理解
* 严格引用原文，标注来源文档 + 页码 + 段落
* 无答案时明确回复 “未找到相关依据”，**不编造、不联想**
* 支持限定范围问答：仅某科室、仅某密级、仅某年份文件

## 3. 政务特色 AI 能力

* **政策对比解读**：新旧政策差异、条款对比
* **高频问题自动汇总**：窗口 / 科室常见咨询问题
* **材料清单智能生成**：根据事项自动生成所需材料
* **敏感内容自检**：涉密、敏感表述、违规用词检测
* **知识过期提醒**：政策废止、时效到期提醒

## 4. AI 权限控制

* 普通人员：仅基础问答
* 涉密岗位：AI 不可读取涉密文档
* 领导岗位：可全局问答 + 数据统计
* AI 对话记录全部留存审计，不可删除

# 四、知识库核心功能（政务内部版）

## 1. 文档库分类

* 政策法规库
* 业务流程库
* 办事指南库
* 会议材料库
* 应急预案库
* 内部管理制度
* 专项工作资料库

## 2. 文档管理

* 目录树 + 标签双管理
* 版本管理（修订记录）
* 归档与销毁流程
* 失效文件标记
* 在线预览不落地

## 3. 检索能力

* 关键词检索
* 文号检索
* 发文单位检索
* 语义检索（私有化向量库）
* 高级筛选：密级、科室、时间、状态

## 4. 协作与流程

* 起草→审核→发布
* 多人协同编辑（内网）
* 评论、批注、督办标记
* 过期政策自动提醒下架

## 5. 数据看板（领导视图）

* 文档总量、更新频次
* 高频访问文件
* AI 问答热点问题
* 各科室知识贡献排行
* 资料完整度统计

***

# 五、技术架构（私有化政务部署）

## 1. 部署方式

* 纯内网 / 专网部署
* 物理机 / 虚拟机 / 政务云容器化
* 无外网访问、无外网调用
* 支持等保 2.0 三级要求

## 2. 技术栈

* 后端：Java SpringBoot（政务主流）
* 数据库：MySQL/PostgreSQL（国产化可选）
* 向量库：Milvus（私有化）/ PGvector
* 对象存储：MinIO（内网）
* 检索：Elasticsearch 内网版
* 大模型：**Qwen/GLM/LLaMA2 私有化量化版**
* 国产化兼容：支持麒麟、统信 UOS

## 3. 安全加固

* 接口鉴权 + IP 白名单
* 传输加密
* 存储加密
* 防 SQL 注入、XSS
* 定期备份、日志持久化
