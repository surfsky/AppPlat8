    /Gis 展示接口来的数据技术方案
        方案一：单独建一个 GisApi 接口表：
            用于管理和监管使用到的所有 API，包括三方提供的；
            字段：Id、MenuId（菜单Id）、DataUrl（数据Url）、DataCnt（最后数据量）、DataDt（最后更新时间）
            窗口：管理页面（Gis/ApiManager，左侧为菜单，右侧为接口）、接口列表(Gis/Apis）、接口表单（Gis/ApiForm）、接口数据（Gis/ApiData）
            接口数据结构：如果是点位，参照GisGeometry + DataResult 封装，如：
                {
                    code: 0,
                    message: 'ok',
                    data : [
                        {..gisgeometry data..},
                        {..gisgeometry data..},
                        ...
                    ]
                }
            接口支持请求参数 region，可传递给接口供过滤数据，如：
                {
                    type : 'Rectangle',           // 矩形范围（如只展示当前地图区域内的对象，以减少地图渲染压力）
                    data : [tlx, tly, brx, bry]   // 左上角和右下角的经纬度
                }
                或
                {
                    type : 'Circle',       // 圆形范围（如搜索某个事件点周边的危险源）
                    center : [lng, lat],   // 中心点经纬度
                    radius : x             // 半径（米）
                }
                或
                {
                    type: 'Polygon',       // 多边形（如搜索某个园区内的企业）
                    data: [.....]          // 多边形数据
                }
                备注：接口数据请求用post、region参数是根据页面及业务需求动态调整的。
                备注：如果数据库原生支持地理位置检索，性能会加强

                - 点击具体点位，右侧滑出“点位详情”面板，显示点位详情；
            优点：独立管理、且接口数据不仅仅可用于图层、也可以用于其它方面（如风雨情等）
            缺点：结构复杂，一个菜单项会对应多个数据接口，无法正确展示数量（可考虑后台统计，并更新到gismenu.datacnt 上以解决此问题）；

            增加参数 DataCnt、DataDt字段
