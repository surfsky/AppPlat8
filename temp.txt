**推荐放在哪个目录**
1. 规格文档放在现有文档目录下的一个新子目录。  
建议新建：Doc/Specs/功能名/  
你仓库里已有 Doc，放这里最自然，也方便团队查阅。

2. 任务拆解可以继续用你现在的 Task.txt，或者每个功能单独一个 tasks.md。  
如果多人协作，建议每个功能目录里单独有任务文件，避免互相覆盖。

3. 测试放在现有测试项目，不新建奇怪位置。  
- 工具/通用逻辑测试放 App.UtilsTests  
- 业务逻辑测试放 App.BLL.Tests

4. 实现代码仍放原分层，不因为 SDD 改结构。  
- 页面与接口： App  
- 业务层： App.BLL  
- UI 组件框架： App.EleUI

**每个功能建议写哪些文件**
在 Doc/Specs/功能名/ 下，建议最少 5 个文件：
1. 01-spec.md  
写业务目标、范围、术语、用户故事、非目标。

2. 02-contract.md  
写输入输出、字段约束、错误码、分页排序规则、兼容性要求。

3. 03-acceptance.md  
写可执行的验收标准，尽量用 Given-When-Then。

4. 04-test-plan.md  
写单元测试清单、集成测试清单、回归点。

5. 05-task-breakdown.md  
写实现任务分解，按顺序和依赖拆成小任务。

**在 VS Code + Copilot 里的工作流**
1. 先写 01-spec.md 和 03-acceptance.md。  
2. 在 Chat 里让 Copilot先审规格完整性，再补 02-contract.md。  
3. 让 Copilot基于 task-breakdown 一次只实现一个小任务。  
4. 每完成一个任务，立即补对应测试并运行。  
5. 全部通过后再做一次“按 acceptance 逐条核对”的 review。

**可直接用的 Copilot 提示语**
1. “请先审阅这个 spec，列出冲突、遗漏、不可测试项，不要写代码。”

4. “根据 acceptance 第 3、4 条补充测试到 App.BLL.Tests，并说明覆盖点。”  
5. “做一次 SDD 对齐检查：代码是否偏离 spec，列出差异和修复建议。”

**你这个项目的实操建议**
1. 从一个清单页功能先试点（例如对象列表筛选/分页这类你最近在改的功能）。  
2. 把分页、筛选、工具栏间距这种“容易回退”的规则写进 acceptance 和 contract。  
3. 在 PR 前固定做一次“规格一致性复核”，减少这类回归。

/// <summary>解析整数参数</summary>
/// <param name="cmdArgs"></param>
/// <param name="key"></param>
/// <param name="defaultValue"></param>
/// <param name="min"></param>
/// <param name="max"></param>
/// <returns></returns>
static int ParseIntArg(string[] cmdArgs, string key, int defaultValue, int min, int max)
{
	var text = cmdArgs
		.FirstOrDefault(t => t.StartsWith(key, StringComparison.OrdinalIgnoreCase))
		?.Substring(key.Length)
		?.Trim();

	if (!int.TryParse(text, out var value))
		return defaultValue;
    if (value < min) return min;
    if (value > max) return max;
	return value;
}

        [HttpApi("获取地址列表（旧）", AuthLogin = true)]
        public static APIResult GetAddrsGeo(string name)
        {
            name = AmapHelper.NormalizeAddressQuery(name);
            if (name.IsEmpty())
                return new APIResult(-1, "请输入地址关键字");

            var list = new List<GisAddrItem>();
            var dedup = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
            var query = new Dictionary<string, string>
            {
                ["address"] = name
            };

            if (!AmapHelper.TryGetAmapResult<AmapGeocodeResult>("/v3/geocode/geo", query, out var geocodeResult, out var failInfo))
                return new APIResult(-1, failInfo);

            if (geocodeResult?.Geocodes != null)
            {
                foreach (var geocode in geocodeResult.Geocodes)
                {
                    var location = AmapHelper.GetJsonText(geocode.Location);
                    if (!GisHelper.TryParseLngLat(location, out var gcjLng, out var gcjLat))
                        continue;

                    var wgs = GisHelper.Gcj02ToWgs84(gcjLng, gcjLat);
                    var formattedAddress = AmapHelper.GetJsonText(geocode.Formatted_address);
                    if (formattedAddress.IsEmpty())
                        formattedAddress = name;
                    var district = AmapHelper.GetJsonText(geocode.District);
                    var key = $"{formattedAddress}:{Math.Round(wgs.Lng, 6)},{Math.Round(wgs.Lat, 6)}";
                    if (!dedup.Add(key))
                        continue;

                    list.Add(new GisAddrItem
                    {
                        Name = formattedAddress,
                        District = district,
                        Address = formattedAddress,
                        Lng = Math.Round(wgs.Lng, 6),
                        Lat = Math.Round(wgs.Lat, 6)
                    });
                }
            }

            return list.Take(12).ToList().ToResult();
        }
